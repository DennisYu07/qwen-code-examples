# Qwen Bolt Demo 使用手册

## 简介

Qwen Bolt Demo 是一个 AI 驱动的在线代码生成平台。你只需要用自然语言描述想要构建的应用，AI 就会自动生成完整的项目代码，并在浏览器中实时运行和预览。

---

## 快速开始

### 方式一：本地开发运行

```bash
# 1. 进入项目目录
cd apps/qwen-bolt-demo

# 2. 安装依赖
npm install

# 3. 启动服务
npm run dev

# 4. 打开浏览器访问
open http://localhost:3000
```

### 方式二：Docker 部署运行

```bash
# 1. 构建镜像
cd apps/qwen-bolt-demo
docker build -t qwen-bolt-demo .

# 2. 运行容器
docker run -d -p 3000:3000 --name qwen-bolt-demo qwen-bolt-demo

# 3. 打开浏览器访问
open http://localhost:3000
```

### 方式三：使用导出的镜像

如果你收到了 `qwen-bolt-demo.tar` 镜像文件：

```bash
# 1. 导入镜像
docker load -i qwen-bolt-demo.tar

# 2. 运行容器
docker run -d -p 3000:3000 --name qwen-bolt-demo qwen-bolt-demo:latest

# 3. 打开浏览器访问
open http://localhost:3000
```

## 功能说明

### 首页

首页是你与 AI 交互的起点，包含以下功能：

#### 输入提示词

在输入框中用自然语言描述你想要构建的应用。例如：

- "帮我创建一个 React Todo 应用"
- "用 Vue 3 做一个天气查询页面"
- "创建一个带登录功能的后台管理系统"

按 **Enter** 或点击 **Build Now** 按钮开始生成。

#### 上传文件/文件夹

点击输入框左下角的 📎 按钮，可以上传已有的项目文件或文件夹。AI 会基于你上传的代码进行修改和优化。

上传后，文件会显示在输入框上方，支持：
- 查看已上传的文件和文件夹
- 点击 ✕ 移除单个文件或整个文件夹

#### 模型选择

点击顶部导航栏的模型选择器，可以切换不同的 AI 模型：
- **Qwen Coder+** — 默认模型，适合大多数场景
- 其他可用模型

#### 模型配置

点击顶部导航栏的 ⚙️ 按钮，可以配置：
- **认证方式** — Qwen OAuth 或 OpenAI API Key
- **API Key** — 如果使用 OpenAI API Key 认证方式
- **Base URL** — 自定义 API 地址
- **模型名称** — 自定义模型标识

#### 主题切换

点击顶部导航栏的 🌙/☀️ 按钮，在亮色和暗色主题之间切换。

#### 语言切换

点击顶部导航栏的语言按钮，在中文和英文之间切换。

---

### 工作区

输入提示词后，会自动跳转到工作区页面。工作区分为三个主要区域：

#### 左侧：聊天区

- **消息列表** — 显示你和 AI 的对话历史
- **附件显示** — 如果消息包含上传的文件，会在消息上方显示文件/文件夹标签
- **输入框** — 继续与 AI 对话，可以追加需求或修改指令
- **文件上传** — 在工作区中也可以随时上传新文件
- **停止按钮** — AI 正在生成时，点击 ⏹ 可以中断生成
- **Token 统计** — 底部显示本次会话的 Token 用量

#### 中间/右侧：代码与预览

通过顶部的 **Code View** / **Preview** 切换按钮，在两种视图之间切换：

##### Code View（代码视图）

- **文件树** — 左侧显示项目的文件目录结构，点击文件可以查看内容
- **代码编辑器** — 右侧显示选中文件的代码，支持语法高亮
- **搜索** — 点击 🔍 按钮可以搜索文件

##### Preview（预览视图）

- 当 AI 生成的项目包含 `package.json` 时，系统会在 AI 对话完成后自动执行 `npm install` 和 `npm run dev`
- 开发服务器启动成功后，预览面板会自动加载应用
- 点击 **Open in New Tab** 可以在新标签页中打开预览

#### 底部：终端面板

- **Terminal** — 集成的命令行终端，可以手动执行命令
- **Output** — 显示系统输出日志
- **多终端** — 点击 **+** 按钮可以创建新的终端 Tab
- **关闭终端** — 非默认终端 Tab 可以通过 ✕ 按钮关闭
- **拖拽调整** — 终端面板顶部边缘可以拖拽调整高度

---

### 聊天历史

点击左侧导航栏的 📋 按钮，可以查看和管理聊天历史：

- 所有对话会自动保存到浏览器本地（IndexedDB）
- 点击历史记录可以恢复之前的对话和代码
- 支持删除单条历史记录

---

### 项目下载

点击顶部的 **Download** 按钮，可以将当前项目的所有文件打包为 ZIP 文件下载到本地。

## 使用技巧

### 提示词编写建议

1. **明确技术栈** — 指定你想使用的框架和库，例如 "用 React + Tailwind CSS"
2. **描述功能细节** — 越详细越好，例如 "包含登录页面、仪表盘、用户列表"
3. **分步迭代** — 先让 AI 生成基础版本，然后通过追加对话逐步完善
4. **上传参考代码** — 如果有现有项目，上传后让 AI 在此基础上修改

### 常见操作

#### 修改已生成的代码

直接在聊天中告诉 AI 你想修改什么：
- "把导航栏改成蓝色"
- "添加一个搜索功能"
- "修复这个 bug：点击按钮没有反应"

#### 上传现有项目进行修改

1. 在首页或工作区点击 📎 按钮
2. 选择项目文件夹上传
3. 在输入框中描述你想要的修改
4. AI 会基于你的代码进行修改

#### 手动运行命令

如果自动启动失败，可以在终端中手动执行：

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

## 常见问题

### Q: 预览页面一直加载不出来？

**A:** 可能的原因：
1. `npm install` 失败 — 查看终端输出，可能是网络问题。可以手动在终端执行 `npm install`
2. 开发服务器未启动 — 在终端手动执行 `npm run dev` 或 `npm start`
3. 浏览器不支持 — WebContainer 需要 Chrome 90+ 或 Edge 90+，Firefox 暂不支持

### Q: npm install 报错？

**A:** 系统已自动配置淘宝 npm 镜像源加速。如果仍然失败，可以在终端手动重试：

```bash
npm install
```

### Q: 上传的文件在代码面板看不到？

**A:** 确保上传的文件不是空文件。上传后文件会立即显示在代码面板的文件树中。

### Q: 聊天历史丢失了？

**A:** 聊天历史保存在浏览器的 IndexedDB 中。清除浏览器数据会导致历史丢失。不同浏览器之间的历史不共享。

### Q: 如何更换 AI 模型？

**A:** 
1. 点击顶部导航栏的模型选择器切换预设模型
2. 或点击 ⚙️ 按钮，选择 "OpenAI API Key" 认证方式，填入你自己的 API Key 和 Base URL

### Q: Docker 部署后无法访问？

**A:** 检查以下几点：
1. 确认容器正在运行：`docker ps`
2. 查看容器日志：`docker logs qwen-bolt-demo`
3. 确认端口映射正确：`-p 3000:3000`
4. 如果在服务器上部署，确认防火墙已开放 3000 端口
